# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Engine & Stack

- **Godot 4.6** with GDScript
- Physics: Jolt Physics
- Renderer: Forward Plus / Direct3D 12 (Windows)

## Running the Project

```bash
godot --run project.godot
# or open project.godot in the Godot editor and press F5
```

There is no separate build, lint, or test step — Godot handles everything through the editor.

## Architecture

Godot uses a scene-node hierarchy. Scenes are `.tscn` files; logic lives in attached `.gd` scripts.

```
game.tscn          ← main scene (set in project.godot → run/main_scene)
└── Player         ← instance of player.tscn
    ├── Sprite2D
    └── CollisionShape2D (CapsuleShape2D)
```

`player.gd` extends `CharacterBody2D` and implements top-down WASD/arrow-key movement at 200 px/s using `move_and_slide()` in `_physics_process(delta)`.

## Godot-Specific Conventions

- Scene composition over inheritance — build entities by combining nodes and scenes.
- `_physics_process(delta)` for movement/collision; `_process(delta)` for visuals/UI.
- Use `Input.is_action_pressed()` with project-defined input actions (Project → Input Map) rather than raw `KEY_*` constants for new input — keeps controls rebindable.
- `.godot/` is auto-generated by the editor; never edit it manually.
